## Real-actionable (fixed in this branch)

- **R1 — U6 e2e native block was missing (correctness P-med/95, testing high/95, reliability med/90 — 3-reviewer agreement).** An earlier `Edit` to `scripts/e2e-eval.sh` silently failed (the anchor text didn't match the file), so the planned native-backend lifecycle block never shipped; the native walker had zero coverage through the real binary. **Fix:** added the `LIFECYCLE (native backend, NER-138 Phase 7 walker)` block — `init → start → save → run → propose → check → accept → restore` on `--content-backend native`, asserting the `forge-tree:` content ref and observable ignore/secret hygiene at the save boundary. e2e went 58 → 70 checks, all green.
- **R2 — env-independence toggles unproven (testing high/90).** Nothing asserted that `git_exclude(false)`/`git_global(false)` are real — a misconfigured toggle would pass every test because the parity corpus had no `.git/info/exclude` entry. **Fix:** added `walk_is_environment_independent_of_info_exclude` — a path excluded only via `.git/info/exclude` is dropped by `git --exclude-standard` but KEPT by the native walk (the load-bearing "native is intentionally more inclusive" direction).
- **R3 — nested-`.gitignore` (R2's headline claim) untested (testing med/80, maintainability noted).** All ignore fixtures were root-level. **Fix:** added `nested_subdir_gitignore_is_honored` (subdir `.gitignore` with a negation, plus a root file outside its scope).
- **R4 — empty-worktree path untested (testing med).** **Fix:** added `empty_worktree_snapshots_and_roundtrips` (empty walk → empty root tree → materialize).
- **R5 — weak divergence assertions (correctness low/60).** The force-added and tracked-then-ignored tests proved the divergence direction but not that it was the *only* difference. **Fix:** added "parity-of-the-rest" `assert_eq!` (strip the index-only path; the rest of the two scans must be equal).
- **R6 — lost defensive dedup (correctness low/70).** The removed `snapshot_candidate_paths` deduped via `BTreeSet`; the new `Vec` path relies on the serial Walk's one-entry-per-path guarantee. **Fix:** added `files.dedup_by(...)` in `scan_worktree` so a future double-yield can't feed `write_tree` two same-named entries; documented the reliance.
- **R7 — submodule + case-fold divergence classes neither tested nor scope-out-commented (testing high/90, plan R5/U3).** The plan committed to "assert-explicitly OR scope-out with an explicit comment." **Fix:** added an explicit scope-out comment block naming both classes and why they're not asserted (case-fold is platform-dependent → would be flaky; submodule needs an offline second-repo fixture → impractical), noting secret hygiene is unaffected (the backstop lowercases names).
- **Cleanup:** removed dead `let paths` / `let _ = paths;` in the special-byte test (maintainability nit); added a non-UTF-8 best-effort doc note to `rel_path` (reliability low/55).

## Defense-in-depth (noted, not blocking)

- **DD1 — DIRTY_WORKTREE guard now rides on walker self-consistency** (reliability med/70). `restore`/`attach` compare a fresh `snapshot_worktree` ref against the stored ref; the dirty check now depends on the walker being stable across calls. No code change for slice 1 (both walks use the same walker, so it's self-consistent); the new e2e `restore` round-trip exercises the double-walk. Flagged so slice 2's base-anchoring work does not silently widen the accepted divergence.
- **DD2 — `map_walk_error` `None`-arm variants (Loop/Glob/Partial) are S1-safe but untested.** Verified path-free by inspection against the `ignore` source; `Loop` is unreachable under `follow_links(false)`. The generic fail-closed message is acceptable; distinguishing variants is marginal.

## Reviewed-and-rejected (explicitly won't fix)

- **RR1 — "confirm the fail-closed walk-error policy is intended" (security P3/25).** It is intended and is the safe direction (aborting a save can never leak a path the backstop would catch; `NotFound`→skip handles the only benign transient). No defect; no change.
- **RR2 — "add a real permission-denied integration test for S1" (testing/reliability low/75).** The unit test hand-constructs the exact `ignore::Error::WithPath{Io(...)}` shape the crate produces and asserts path-freeness in both `to_string()` and `{:#}`. A real `chmod 000` test is flaky/perms-dependent (bypassed under root/CI) and adds little over the deterministic unit proof. Deliberate gap.
- **RR3 — "thread `OsStr`/`PathBuf` through `scan_worktree` for faithful non-UTF-8 capture" (reliability low/55).** Out of slice-1 scope; the prior git `.lines()` parsing was also lossy for such names, so this is not a regression. Documented as a known limitation in `rel_path`; the secret backstop is unaffected.
