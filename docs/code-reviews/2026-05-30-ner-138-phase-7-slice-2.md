## Defer-able — filed/noted for slice 3

- **Commit-DAG integrity in `doctor`** (cycle / dangling-parent detection): `verify_reachable` traverses only `Tree` objects, so a `Commit` whose `tree`/parents are missing surfaces only at read time, not via `doctor`. This is exactly the whole-phase exit criterion "the DAG has no cycles/dangling parents (doctor verifies)" — slice 3, where navigation makes the DAG load-bearing. (reliability P3, adversarial residual). Carried in the slice-3 handoff.
- **`evidence_digest` write-time guard**: the field is structurally always `None` in slice 2 (genesis only), so there is no reachable leak in this diff. When slice 3 populates it at accept, the populating code must hash the evidence `content_hash` and never assign excerpt text (the commit payload bypasses `redact_evidence_excerpt`). Security reviewer suggests a `Hex64` newtype / `debug_assert` then. Carried in the slice-3 handoff.
- **`BranchExists`-on-native-base-with-changed-tree** test (testing P3): the idempotent-reconcile path is covered; the mismatch path is symmetric and low-risk. Cheap to add in slice 3 when commit-on-accept exercises more export shapes.

## Defense-in-depth — noted, not blocking

- **Cross-machine synthesized-parent determinism**: the within-environment reconciliation case (the load-bearing one for idempotent re-export) is fully deterministic; the autocrlf pin (fix #5) closes the line-ending vector. Cross-git-version drift (e.g. a future sha256-default git) remains a documented residual, revisited when git export is demoted to optional interop (slice 3). The plan's Risk table records it.
- **Identical-worktree repos → identical genesis commit id** (the format omits actor/timestamp from the hashed bytes): harmless under the per-repo model (`base_head` is namespaced by the repo's own DB + object store, never a cross-repo key). Flagged for Phase 9 sync/signing identity. The slice-3 design note already requires actor + authored-time in the first *justified* commit format.
- **`all_object_ids` three-kind re-hash** (now `[Blob, Tree, Commit]`): ~1.5× the prior per-object hashing on the diagnostic gc/doctor path. Documented; slice-3 object-kind headers remove the multi-hash scan.
- **`synthesize_deterministic_commit` / git plumbing have no subprocess timeout**: pre-existing pattern across `forge-export-git`; not introduced here.

## Reviewed-and-rejected — false positives / intentional

- **`flatten_tree_into` infinite recursion on a cyclic tree** (correctness residual): not constructible. Native trees are content-addressed and built bottom-up by `write_tree`; `read_object` hash-verifies every load, so a self-referential tree id is cryptographically infeasible. Same recursion shape as the pre-existing `materialize_tree`/`write_tree`/`verify_reachable`. **Rejected** (not a slice-2 regression).
- **Genesis captures a mid-`save` dirty tree** (adversarial A9 / correctness testing gap): verified the lifecycle invariant holds — `save` requires an active attempt that only `start` creates; `start` calls `current_base` (→ `ensure_head`) first; every `current_base` caller holds the advisory lock (`is_mutating_command` covers them; `run` never calls them). Genesis is always taken at start-time. **Rejected as a bug**; recorded as a load-bearing invariant in the plan/handoff (a future command that snapshots before a start would need to re-establish this).
- **`commit.gpgsign` breaking synthesized-parent determinism**: `git commit-tree` does not honor `commit.gpgsign` (only `git commit` does), so a repo-local `commit.gpgsign=true` does not non-determinize or prompt. **Rejected** (verified).
- **`base_head` value-shape change breaks consumers** (api-contract P1): this is the intended, documented contract change — `base_head` is opaque `TEXT`, and native repos now emit `f1:commit:` instead of a git SHA. No in-repo consumer parses it as a git SHA (stale-base is a string compare; export resolves via the `ObjectId::parse` discriminator; `compare`/`verify-branch` treat it opaquely). **Reviewed-and-accepted** as the slice's load-bearing reversal, documented in the plan; the `forge schema` docs note `base_head` is opaque.
