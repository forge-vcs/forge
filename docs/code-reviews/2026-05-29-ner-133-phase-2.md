## Real-actionable findings ‚Äî fixed on the branch (`f2c36c5`)

1. **P0/P1 ‚Äî migration reconciliation brick.** A DB created by the `cd1bb3b`-era binary (on `main`) sits at `MAX(version)=1` with `content_backend` **inline** (its `001` had the column; version 2 did not yet exist) but without `attached_attempt_id`. The numbered runner ran `002`'s `ALTER ADD COLUMN content_backend` ‚Üí `duplicate column name` ‚Üí raw error ‚Üí `COMMAND_FAILED`; since `migrate()` runs every command, the repo bricked. The convergence test's "v1" used a baseline shape (neither column) no shipped binary produced. **Fix:** apply each migration **statement-by-statement**, tolerate `duplicate column name` (additive ADD COLUMNs already satisfied ‚Üí converge), wrap other DDL failures as `MIGRATION_FAILED`. Test `cd1bb3b_v1_with_inline_content_backend_converges` added. (adversarial; data-migrations corroborated the FK/atomicity concerns.)
2. **P1 ‚Äî deferred-finding #1 unfixed on the production path.** Every mutating command calls `insert_operation_view`, whose singleton-CAS guard raised plain `anyhow!("current operation changed")` ‚Üí recorded under `--request-id` (poisoning it). The typed retryable `CurrentStateChanged` lived only in the unused `create_operation_view`. **Fix:** `insert_operation_view` now raises `CurrentStateChanged` (CONFLICT, retryable) ‚Üí not recorded ‚Üí retry re-executes; dead `create_operation_view` removed; the `run` child-re-execution caveat documented in the schema `notes`. (adversarial, api-contract, testing, correctness.)
3. **P2 ‚Äî `record_conflict_set` masked `STALE_BASE`.** `record_conflict_set(...)?` before raising `StaleBase` meant a metadata-insert failure surfaced as `COMMAND_FAILED`. **Fix:** best-effort (`let _ = ‚Ä¶`) at both sites. (correctness, reliability, maintainability ‚Äî 3-reviewer agreement.)
4. **P2 ‚Äî replayed failures dropped `errors[].details`.** `record_failed_operation` stored only `{message, code}`; `replay_response` rebuilt the error without details, contradicting the published `details_keys`. **Fix:** store + restore `details` in `error_json`. (api-contract.)
5. **P2 ‚Äî `MIGRATION_FAILED` advertised but unreachable on DDL failure.** Only the checksum-mismatch branch produced it. **Fix:** folded into #1's per-statement wrapping. (adversarial.)
6. **P2/P3 ‚Äî three drifting `50ms` backoff constants; native `git add -A` stored secret blobs.** **Fix:** single `forge_protocol::RETRY_BACKOFF_MS`; `synthesize_git_tree` deletes secret-risk files before `git add -A`. (maintainability, api-contract, reliability.)
7. **Testing gaps closed:** CLI-code drift guard, replay-details parity, native secret-blob exclusion, at-HEAD-drift-surfaced-not-repaired, CAS classification.

- **Fold `LockTimeout` into `ForgeError`** (maintainability M-04). The drift risk (after_ms parity, registry coverage) is closed by the shared constant + the CLI-code drift-guard test; the structural fold is cleanup. Deferred.
- **`migrate()` reuses the pre-lock connection** (reliability REL-003). Benign (idempotent apply, WAL keeps the fd, advisory lock is the same `forge.lock`); re-opening after acquiring the lock is defense-in-depth. Deferred.
- **`bootstrap_schema_migrations` DDL not atomic** (data-migrations P2). `CREATE TABLE` + `ALTER ADD checksum` run in autocommit; a crash between them is self-healing on the next call. Deferred as hardening.
